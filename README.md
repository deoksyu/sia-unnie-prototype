# 시아언니 — 웹 프로토타입

'사브작 아카이브 시아언니' 기획서 기반 클릭 가능 프로토타입입니다. 순수 HTML/CSS/JS 단일 파일이라 별도 설치 없이 `index.html`을 브라우저로 열면 바로 확인할 수 있습니다.

## 실행 방법
그냥 더블클릭해서 브라우저로 열면 됩니다. (또는 `npx serve .` 등 아무 정적 서버로 열어도 됩니다)

## 지금 들어있는 것 (목업 기준)
- 온보딩(전화번호 인증): 현재는 스킵 처리되어 앱을 열면 바로 랜딩(시아언니 홈) 화면이 나옵니다. 인증 화면 코드/로직 자체는 `renderOnboarding()`에 그대로 남아있고, `STATE.onboard` 초기값만 `'intro'`에서 `'done'`으로 바꿔둔 상태라 나중에 되돌리기 쉽습니다.
- 홈(시아언니 대화): 대화형 기록, 위기 키워드 감지 시 상담 자원 안내, 일정 대화 후 편지 제안
- 나의 회복지도: 엑셀 같은 격자(매트릭스) UI. 가로축 = 치료 8단계(진단/검사/수술/항암/방사선/회복기/정기검사/유지치료) 고정 8열, 세로축 = 시간(월→주→일, 최신이 맨 위) — 세로 스크롤이 기본 탐색 방식입니다. 월 단위 전체 보기에서 시작해 시간 행(맨 왼쪽 라벨)이나 칸을 누르면 그 달의 주 단위로, 다시 누르면 그 주의 일 단위로 액자식으로 확대됩니다. 확대는 클릭/탭 또는 두 손가락 핀치(세로 방향)로 가능합니다 — 1손가락 스크롤과 제스처가 겹치지 않도록 마우스 휠로 확대하는 기능은 이번에 뺐습니다(휠은 이제 그냥 스크롤만 합니다). 일(day) 레벨의 칸을 누르면 그 시점·그 단계의 기록을 바로 쓰고 저장할 수 있는 모달이 열리고(기존 기록이 있으면 함께 보여줌), 저장된 내용은 '내 기록' 탭에도 동일하게 반영됩니다. 뒤로가기(←)로 일→주→월 순으로 다시 축소할 수 있습니다.
- 사브작 편지함: 보낸 편지(답장 대기/도착), 받은 편지함(100자 이내 답장 작성 + 간단 금칙어 필터)
- 시아언니의 참견타임: 상태요약 / 작은 실천 / 의료진 문의 권유 3단 구성
- 내 기록: 대화에서 쌓인 원본 기록 리스트

모든 데이터는 새로고침하면 초기화되는 인메모리 목업입니다 (`DB` 객체, `index.html` 상단).

가로 스크롤 영역(타임라인, 단계 칩, 빠른 입력 칩)은 터치와 마우스 양쪽 다 손가락/커서로 눌러서 끄는 드래그 스크롤을 지원합니다(Pointer Events 기반, `index.html` 하단 `dragState` 로직). 실제로 스크롤 없이 드래그만 한 경우엔 뒤따르는 탭/클릭을 무시해서 오작동(의도치 않은 화면 이동)을 막았습니다.

모바일에서 스크롤 시 화면이 당겨서 새로고침되는 것처럼 보이는 문제 대응: `overscroll-behavior`는 Android/Chrome에는 잘 먹히지만 iOS/WebKit에서는 무시되는 경우가 많아, `html/body`를 아예 `position:fixed`로 고정해 페이지 자체가 스크롤/바운스될 여지를 없앴습니다. 실제 스크롤은 전부 내부 `.screen`, `.hscroll`, `.tl-wrap` 안에서만 일어납니다.

확대/축소 슬라이더를 움직일 때마다 화면이 깜빡이며 새로고침처럼 보이던 문제도 별개 원인이었습니다 — 줌 값이 바뀔 때마다 앱 전체를 `render()`로 다시 그리면서 슬라이더 `<input>` 자체가 매번 새 DOM 노드로 교체돼, 브라우저의 네이티브 드래그 트래킹이 끊기는 것이었습니다. `zoomTimelineTo()`가 더 이상 전체 `render()`를 호출하지 않고 타임라인 내부(`#tlScroll`)와 퍼센트 라벨만 직접 갱신하도록 바꿨고(`updateTimelineDOM()`), 슬라이더/버튼 DOM은 절대 재생성되지 않습니다. 타임라인 클립은 줌마다 새로 그려지므로 클립 클릭은 개별 바인딩 대신 전역 위임(delegated click)으로 처리해 줌 이후에도 항상 동작합니다.

## 다음 단계 — 실제 연동 시 확인할 지점

### 1. Claude API 연동
`index.html`의 `getSiaReply()` 함수 안에 표시된 지점입니다.
```js
const CONFIG = {
  USE_MOCK_AI: true, // 실제 연동 시 false
  ANTHROPIC_API_KEY: '', // 프런트엔드에 직접 넣지 말 것
  ...
};
```
API 키를 브라우저 JS에 그대로 넣으면 노출됩니다. 실제 연동은 Supabase Edge Function 같은 서버리스 프록시를 하나 두고, 프론트는 그 프록시만 호출하는 구조를 권장합니다. 키를 전달해주시면 이 프록시 함수와 연동 코드를 이어서 작성하겠습니다.

### 2. Supabase 스키마 (제안)
```
users            id, phone, created_at
records          id, user_id, stage, date, text, tags[], mood, source
sent_letters     id, user_id, record_id, text, status(pending/replied), created_at
letter_replies   id, letter_id, replier_user_id, text, created_at
inbox_matches    id, letter_id, recipient_user_id, replied(bool)
```
`records.stage`는 기획서의 8단계(진단/검사/수술/항암/방사선/회복기/정기검사/유지치료) enum으로 관리하면 회복지도 쿼리가 단순해집니다.

### 3. 위기 대응
`CONFIG.CRISIS_KEYWORDS`, `CONFIG.CRISIS_RESOURCES`가 지금은 클라이언트 키워드 매칭입니다. 실서비스에서는 이 감지를 서버 단(레코드 저장 시점)에서도 한 번 더 돌려, 클라이언트를 우회해도 걸리도록 이중화하는 걸 권장합니다.

### 4. 편지 매칭 로직
지금은 받은 편지함이 고정 목업입니다. 실제로는 "비슷한 시기·비슷한 단계의 사용자"를 어떻게 정의할지(치료 단계 일치, 태그 유사도, 활동 시간대 등) 정책이 필요합니다.

### 5. 랜딩 화면
기획서 5페이지 실제 디자인 스크린샷을 기준으로 CSS/SVG로 재현했습니다. 1차 시도가 실제 화면과 많이 달라 캐릭터를 처음부터 다시 그렸습니다: 긴 웨이브 헤어(네이비 단색 채움)에 동그란 얼굴, 볼터치, 미소, 어깨 옆 반짝임 표시가 있는 여성 캐릭터(`SIA_CHARACTER_SVG`), 잎이 쌍으로 뻗은 식물 일러스트(`SIA_PLANT_SVG`), 우측 상단 알림 종 아이콘(`SIA_BELL_SVG`) 추가. 하단 태그 칩(잠/몸/마음/검사)은 각각 파스텔 배경색(연블루/연핑크/연그린/연옐로)을 갖도록 수정했고, 하단 탭바 아이콘도 이모지 대신 집/지도/편지봉투/사람 형태의 라인 아이콘(`NAV_ICONS`)으로, 활성 탭 색상은 올리브 그린(#4E6142)으로 바꿨습니다. 입력창은 배경 버튼 없이 마이크 아이콘만 보이도록 단순화했습니다. 칩을 누르면 `CHIP_PHRASES`에 매핑된 문장이 입력창에 자동으로 채워집니다.

이 세션에서는 이미지를 직접 렌더링해서 스크린샷으로 대조하는 것이 불가능해(파일 뷰어 접근 제한), 원본 스크린샷을 좌표/색상 단위로 분석해 코드로 재현했습니다. 실제 화면에서 비교 후 어긋난 부분이 있으면 알려주시면 바로 수정하겠습니다.

## 알려진 수정 이력
- 매트릭스 칸(.mx-cell)에 명시적 width가 빠져 있어 헤더(3월/4월…)와 실제 칸이 어긋나 보이던 정렬 버그를 수정했습니다. 이제 헤더·칸·행 전체가 같은 폭 기준으로 정확히 맞습니다.
- `#mxScroll`가 `.hscroll`의 기본 `touch-action:pan-x`를 그대로 물려받아 세로 스크롤(overflow-y:auto)이 터치에서 막혀 있던 문제를 `#mxScroll { touch-action:pan-y }`로 분리해 해결했습니다.
- 랜딩(홈) 화면을 기획서 실제 디자인 스크린샷 기준으로 다시 그렸습니다: 캐릭터/식물 인라인 SVG, 말풍선, 아이콘 칩, 마이크 입력창 추가. jsdom으로 렌더링·칩 클릭·메시지 전송 동작을 모두 재검증했습니다.
